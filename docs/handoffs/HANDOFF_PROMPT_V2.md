## 2. ARCHITECTURE SNAPSHOT

### Data Model (`lib/data.ts`)

**Critical:** The `Property` interface has **two sets of field names**:
- **Canonical** (DB-aligned): `areaSqft`, `localityName`, `cityName`, `propertyTypeName`, `listingIntentId`, `isReraVerified`
- **Legacy aliases** (optional, populated by `mkProperty()`): `area`, `locality`, `city`, `type`, `isVerified`, `listingType`, `purpose`, `furnishing`, `status`, `bedrooms`, `address`

When writing NEW code, always use canonical names. Legacy aliases exist only to keep older components compiling.

**Lookup IDs:**
- `listingIntentId`: 1=buy, 2=rent, 3=pg
