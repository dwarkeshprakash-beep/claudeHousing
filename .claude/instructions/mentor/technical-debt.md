# Technical Debt Tracking

This document tracks known technical debt and areas for improvement in the ApnaNest project.

## High Priority

- **Testing Coverage**: Increase unit test coverage for the Application layer in the backend.
- **Error Handling**: Standardize error response formats across all API endpoints.
- **Documentation**: Keep instruction files and `CLAUDE.md` updated as the project evolves.

## Medium Priority

- **Performance**: Optimize expensive database queries using indexes or caching.
- **Refactoring**: Extract common UI patterns into reusable components in `components/common/`.
- **API Versioning**: Implement a versioning strategy for the API.

## Low Priority

- **Logging**: Enhance structured logging with more contextual information.
- **CI/CD**: Fully automate the deployment pipeline.
- **i18n**: Expand Hindi translations for all pages.

## How to Contribute

When you encounter technical debt:
1. Fix it if it's small and relevant to your current task.
2. If it's large, document it here with a description of the issue and a proposed solution.
3. Discuss with the team during planning.
